एक्सआरपी लेजर पर रचनाकारों के स्वामित्व वाली रिलीज़ प्रणाली। कार्य जारी करें, सीधे बेचें, संग्राहकों के लिए लाभ अनलॉक करें, राजस्व का प्रबंधन करें - ये सभी टिकाऊ ऑन-चेन प्रमाणों द्वारा समर्थित हैं।

> **प्रारंभिक रिलीज़।** RC.2 एक टेस्टनेट पूर्वावलोकन उत्पाद है। इंजन आर्किटेक्चर टेस्टनेट और मेननेट दोनों का समर्थन करता है, लेकिन सभी ट्रस्ट प्रमाणों को केवल टेस्टनेट पर सत्यापित किया गया है। मेननेट एक सुरक्षित और सावधानीपूर्वक मार्ग है - यह डिफ़ॉल्ट नहीं है।

## इसका उपयोग करने के दो तरीके:

### डेस्कटॉप ऐप (रचनाकारों के लिए अनुशंसित)

विंडोज इंस्टॉलर को [GitHub रिलीज़](https://github.com/mcp-tool-shop-org/xrpl-creator-capsule/releases/tag/v1.0.0-rc.2) से डाउनलोड करें और [शुरुआती गाइड](https://mcp-tool-shop-org.github.io/xrpl-creator-capsule/handbook/beginners/) का पालन करें।

**स्टूडियो मोड** आपको एक निर्देशित 6-चरणीय प्रक्रिया के माध्यम से मार्गदर्शन करता है:

1. अपनी रिलीज़ का वर्णन करें (शीर्षक, कलाकार, संस्करण का आकार, फ़ाइलें)
2. संग्राहकों के लाभ निर्धारित करें (बोनस ट्रैक, स्टेम, उच्च-रिज़ॉल्यूशन कला)
3. नियमों और सुरक्षा की समीक्षा करें
4. XRPL टेस्टनेट पर प्रकाशित करें
5. संग्राहकों की पहुंच का परीक्षण करें
6. एक रिकवरी बंडल उत्पन्न करें

[Node.js 22+](https://nodejs.org/) की आवश्यकता है (भविष्य में एक रिलीज़ में बंडल रनटाइम उपलब्ध होगा)।

### CLI (डेवलपर्स और इंटीग्रेटर्स के लिए)

```bash
git clone https://github.com/mcp-tool-shop-org/xrpl-creator-capsule.git
cd xrpl-creator-capsule
npm install
bash verify.sh    # 265 tests, zero network calls
```

15 कमांड जो पूरे रिलीज़ चक्र को कवर करते हैं:

| आदेश। | उद्देश्य। |
|---------|---------|
| `init-wallets` | एक "जारीकर्ता" (issuer) और "संचालक" (operator) के लिए वॉलेट का जोड़ा बनाएं और उसके लिए धन उपलब्ध कराएं। |
| `configure-minter` | "एसेट ऑपरेटर को जारीकर्ता खाते पर अधिकृत 'मिंटर' के रूप में सेट करें।" |
| `create-release` | एक मैनिफेस्ट इनपुट फ़ाइल से एक रिलीज़ बनाएं। |
| `validate` | एक रिलीज़ मैनिफेस्ट को स्कीमा के अनुसार सत्यापित करें। |
| `resolve` | जांच करें कि क्या मैनिफेस्ट पॉइंटर संरचनात्मक रूप से सही हैं। |
| `mint-release` | मिंट एनएफटी (NFT) संस्करण और जारी करने की रसीद (रिसीट) जारी करना। |
| `verify-release` | "मैनिफेस्ट (माल सूची) और रसीद की तुलना श्रृंखला की स्थिति से करें।" |
| `create-access-policy` | एक एक्सेस पॉलिसी (पहुंच नीति) को मैनिफेस्ट (manifest) और रसीद (receipt) से उत्पन्न करें। |
| `grant-access` | पहुंच के अनुरोध का मूल्यांकन करें और अनुमति की पावती जारी करें। |
| `recover-release` | "आर्टिफैक्ट्स और चेन स्टेट का उपयोग करके एक रिलीज को फिर से बनाना।" |
| `create-governance-policy` | एक "रिलीज़ ट्रेज़री" के लिए एक शासन नीति तैयार करें। |
| `propose-payout` | एक शासन नीति के आधार पर भुगतान प्रस्ताव तैयार करें। |
| `decide-payout` | अनुमोदन प्राप्त करें और एक निर्णय पावती जारी करें। |
| `execute-payout` | भुगतान प्रक्रिया का रिकॉर्ड रखें और हैश श्रृंखला की पुष्टि करें। |
| `verify-payout` | सभी 4 शासन संबंधी दस्तावेजों और उनके आपसी संबंधों की जांच करें। |

## यह क्या साबित करता है।

XRPL क्रिएटर कैप्सूल, एक्सआरपी लेजर को रचनात्मक कार्यों के स्वामित्व, भुगतान, पहुंच और सुरक्षा के लिए एक मजबूत नियंत्रण प्रणाली के रूप में उपयोग करता है। यह कोई बाजार नहीं है - यह वह बुनियादी ढांचा है जो बाजारों को वैकल्पिक बनाता है।

## आर्किटेक्चर।

```
app/              Desktop app (Tauri v2 + React)
  src/            Studio Mode + Advanced Mode UI
  src-tauri/      Rust backend (file I/O, bridge dispatch)
  bridge-worker   Engine bridge (stdin/stdout JSON-RPC)
packages/
  core/           Canonical contracts, schemas, validation, hashing
  xrpl/           XRPL client (connect, mint, verify, holder checks)
  storage/        Content store + delivery provider interfaces
  xaman/          Wallet-mediated signing via Xaman
  cli/            15 CLI commands
artifacts/        Live Testnet proof artifacts
site/             Handbook (Astro Starlight)
```

5 इंजन पैकेजों + डेस्कटॉप ऐप वाला मोनोरेपो। टाइपस्क्रिप्ट, विटेस्ट, टाउरी v2, Node 22+।

## नेटवर्क स्थिति

यह प्रणाली नेटवर्क के बारे में पूरी जानकारी रखती है - टेस्टनेट और मेननेट अलग-अलग, कॉन्फ़िगर करने योग्य लक्ष्य हैं।

| | टेस्टनेट | मेननेट |
|-|---------|---------|
| **Default** | हाँ | No |
| **Trust-proven** | हाँ (लाइव प्रमाण, 265 परीक्षण) | अभी नहीं |
| **CLI guard** | किसी की आवश्यकता नहीं | `--network mainnet --allow-mainnet-write` |
| **Desktop app** | स्टूडियो मोड डिफ़ॉल्ट | RC.2 में उपलब्ध नहीं |

**इस रिलीज़ को टेस्टनेट पूर्वावलोकन के रूप में मानें।** आर्किटेक्चर केवल टेस्टनेट तक सीमित नहीं है, लेकिन ट्रस्ट प्रमाण टेस्टनेट पर सत्यापित है। मेननेट के लिए लाइव ज़ामाइन हस्ताक्षर और जानबूझकर प्रचार की आवश्यकता होती है - यह कोई डिफ़ॉल्ट सेटिंग नहीं है।

## विश्वास मॉडल।

**यह प्रणाली किन चीजों से संबंधित है:**
- स्थानीय JSON फाइलें (मैनिफेस्ट, रसीदें, नीतियां, बंडल)
- XRPL (एक्सआरपी लेजर प्रोटोकॉल) वेबसॉकेट के माध्यम से (`wss://`) नए टोकन बनाने, सत्यापन करने और धारकों की जांच करने के लिए।
- वॉलेट के "सीड वाक्यांश" स्थानीय `wallets.json` फ़ाइल में संग्रहीत होते हैं (जिन्हें गिट द्वारा अनदेखा किया जाता है और कभी भी कमिट नहीं किया जाना चाहिए)।

**यह प्रणाली निम्नलिखित पहलुओं को शामिल नहीं करती:**
- एक्सआरपीएल नोड्स के अलावा कोई भी बाहरी एपीआई (API)
- कोई भी डेटाबेस, क्लाउड स्टोरेज या तृतीय-पक्ष सेवाएं
- कोई भी उपयोगकर्ता विश्लेषण, ट्रैकिंग या टेलीमेट्री (डेटा संग्रह)

**सुरक्षा सीमाएं:**
- मुख्य नेटवर्क पर लिखने के लिए `--network mainnet --allow-mainnet-write` का स्पष्ट रूप से उपयोग करना आवश्यक है।
- वॉलेट की जानकारी स्थानीय रूप से संग्रहीत रहती है और केवल लेनदेन पर हस्ताक्षर करने के लिए XRPL को भेजी जाती है।
- सभी हैश, `sortKeysDeep()` नामक एक निश्चित प्रक्रिया का उपयोग करके SHA-256 एल्गोरिदम पर आधारित होते हैं।
- प्रत्येक फ़ाइल (आर्टिफैक्ट) को स्वतंत्र रूप से लेज़र के विरुद्ध सत्यापित किया जा सकता है।
- `xrpl` का संस्करण 4.2.5 पर निश्चित किया गया है (npm आपूर्ति श्रृंखला सलाह के बाद)।
- कोई भी डेटा एकत्र या भेजा नहीं जाता है।

## ज्ञात सीमाएँ

- डेस्कटॉप ऐप के लिए **Node.js की आवश्यकता है** (बंडल रनटाइम जल्द ही उपलब्ध होगा)
- **ज़ामाइन QR हस्ताक्षर अभी लाइव नहीं हैं** - वॉलेट क्रेडेंशियल फ़ाइल की आवश्यकता है (सीड-आधारित, केवल टेस्टनेट)
- **IPFS अपलोड लंबित है** - फ़ाइल पॉइंटर स्थानीय पथ का उपयोग करते हैं, वास्तविक कंटेंट-एड्रेस्ड स्टोरेज जल्द ही उपलब्ध होगा
- **केवल विंडोज** - macOS इंस्टॉलर भविष्य के RC में योजनाबद्ध है
